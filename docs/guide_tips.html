7. 活用テクニック 7. Tips: Useful Techniques

本ツールの「反映」ボタンが手動である理由と、その仕様を活用した高度なテクニックについて解説します。 This section explains why the "Reflect" button is manual and advanced techniques utilizing this specification.

🔒 手動同期のメリット (Safety Design) 🔒 Benefits of Manual Synchronization (Safety Design)
本ツールは、誤操作によるデータの破損を防ぐため、自動同期を行いません。各エディタは「反映」ボタンを押すまで互いに干渉せず、独立して動作します。 This tool does not auto-sync to prevent accidental data corruption. Each editor operates independently and does not interfere with the others until the "Reflect" button is pressed.

🛠️ 3つの活用パターン 🛠️ Three Usage Patterns

🧹 1. プロンプトの正規化 🧹 1. Prompt Normalization
表記揺れや全角文字を含むプロンプトを、標準的な形式に自動整形（クリーニング）します。 Automatically formats (cleans) prompts containing notation variations or full-width characters into a standard format.

  1. テキストエディタにそのまま貼り付けます。 Paste directly into the Text Editor.
  2. ⬇️ ボタンでビジュアルエディタに反映。 button to reflect in the Visual Editor.
  3. 自動的にパースされ、クリーンなタグになります。 It is automatically parsed into clean tags.
🧪 2. 非破壊的な試行錯誤 🧪 2. Non-destructive Editing
元のデータを保持したまま、タグの削除や並べ替えを安全にテストできます（サンドボックス機能）。 Safely test deleting or reordering tags while preserving the original data (Sandbox feature).

  • 失敗した場合: If failed:
    再度 ⬇️ を押すとリセット（Undo）されます。 Press ⬇️ again to reset (Undo).
  • 成功した場合: If successful:
    ⬆️ を押して変更を確定（保存）します。 Press ⬆️ to confirm (save) changes.
🔄 3. データ形式の相互変換 🔄 3. Data Format Conversion
ビジュアルエディタを経由することで、「テキスト形式」と「スプレッドシート（表）形式」を相互に変換できます。 By bridging through the Visual Editor, you can mutually convert between "Text format" and "Spreadsheet (Table) format".

変換したいデータを貼り付け → ⬇️ → タブ切り替え → ⬆️ Paste data → ⬇️ → Switch Tab → ⬆️
